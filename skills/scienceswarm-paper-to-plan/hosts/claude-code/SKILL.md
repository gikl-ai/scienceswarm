# ScienceSwarm Paper To Plan

Use this skill when a source document needs to become an executable plan rather
than a passive summary.

## Workflow

1. State the target outcome in one sentence.
2. Enumerate prerequisites, dependencies, and hidden assumptions.
3. Break the work into ordered phases with concrete deliverables.
4. Call out the most likely failure modes and blockers.
5. Define the fastest validation step that proves the plan is on track.

## Output shape

- `Outcome`
- `Prerequisites`
- `Phases`
- `Failure modes`
- `Fastest validation`
