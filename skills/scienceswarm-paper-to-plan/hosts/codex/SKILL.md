# ScienceSwarm Paper To Plan

Use this skill when you need to turn a paper or technical artifact into an
implementation or reproduction plan that can survive contact with reality.

## Workflow

1. Define the target output and success condition.
2. Extract prerequisites, dependencies, and assumptions from the source.
3. Produce an ordered plan with explicit checkpoints and owners.
4. Identify where the plan is most likely to fail.
5. End with the first validation step that should run before scaling effort.

## Output shape

- `Target output`
- `Prerequisites`
- `Plan`
- `Failure modes`
- `First validation`
